# Pokémon Damage Calculator - macOS App

A lightweight native macOS application for the Pokémon damage calculator, powered by PyQt5 and Flask.

## Features

- **Native macOS Application** — Runs as a standalone .app bundle
- **Embedded Web UI** — Uses the existing HTML/JS calculator UI in a WebView
- **mGBA Script Integration** — Import Pokémon team data from mGBA lua scripts via UI dialog
- **Lightweight** — Minimal resource footprint compared to Electron alternatives

## Development Setup

### Prerequisites

- Python 3.9 or later
- macOS 10.13 or later
- Node.js 14+ (for building the calculator)

### Installation

1. Install Python dependencies:
   ```bash
   pip install -r macos-app/requirements.txt
   ```

2. Build the calculator (from root directory):
   ```bash
   npm install
   npm run build
   ```

### Running Locally

1. Start the application:
   ```bash
   python macos-app/src/app.py
   ```

2. The app will:
   - Launch a PyQt5 window
   - Start a Flask web server on `localhost:5000`
   - Load the calculator UI in an embedded WebView

3. Use **Tools > Import mGBA Script Data** to import team data

## Building the macOS App Bundle

1. Ensure the calculator is built:
   ```bash
   npm run build
   ```

2. Build the .app bundle:
   ```bash
   python macos-app/setup.py py2app
   ```

3. The app will be created at `dist/Pokémon Damage Calculator.app`

4. Move to Applications folder:
   ```bash
   cp -r dist/Pokémon\ Damage\ Calculator.app /Applications/
   ```

5. Launch from Spotlight or Applications folder

## Architecture

```
macos-app/
├── src/
│   ├── app.py              — Main PyQt5 application window
│   ├── server.py           — Flask web server
│   └── script_import_dialog.py  — mGBA script import UI
├── dist/                   — Built web UI (generated by npm build)
├── requirements.txt        — Python dependencies
└── setup.py               — Packaging configuration (py2app)
```

### Data Flow

1. **User opens app** → PyQt5 window launches
2. **Flask server starts** → serves HTTP://localhost:5000
3. **WebView loads calculator UI** → existing HTML/JS works unchanged
4. **User imports script data** → Parse text → Inject into web UI via JavaScript
5. **Calculator functions** → Same as web version (calls compiled TypeScript)

## mGBA Script Integration

The app expects mGBA lua script output in one of these formats:

### JSON Format (Preferred)
```json
{
  "team": [
    {
      "species": "Pikachu",
      "level": 50,
      "moves": ["Thunderbolt", "Quick Attack"],
      "ability": "Static",
      "nature": "Timid",
      "item": "Leftovers"
    }
  ]
}
```

### Text Format
```
Pokemon: Pikachu
Level: 50
Move1: Thunderbolt
Move2: Quick Attack
Ability: Static
Nature: Timid
Item: Leftovers
```

See `script_import_dialog.py` for the parser implementation.

## Troubleshooting

**"Calculator not built" error**
- Run `npm run build` from the root directory

**Flask port already in use**
- Change port in `src/app.py` and `src/server.py`

**JavaScript bridge not working**
- Ensure the web UI is fully loaded before importing data
- Check browser console for errors (Cmd+Option+I in WebView)
